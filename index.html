What This Project Does

This project implements a local-first AI data analyst for exploring CSV, XLSX, and Parquet files directly from your own machine. A user selects a dataset, asks an analytical question, and receives a structured answer backed by DuckDB queries, local model narration, and persisted evidence.

How It Works

  • Local dataset → CSV, XLSX, or Parquet file
  • DuckDB → local profiling and analytical queries
  • data_analyst agent → analytical reasoning layer
  • Ollama → local LLM narration with deepseek-r1:8b
  • Artifacts → traceable results, tables, charts, and run outputs

Architecture Overview

Interfaces

Web UI, local API, and CLI share the same application core.

Runtime

Runs are coordinated through a clear runtime layer with local state and artifacts.

Adapters

DuckDB and Ollama are isolated behind adapters to keep the core clean.

Engineering Highlights

AI-Assisted Engineering

The repository keeps AGENTS.md visible because AI assistance was used inside explicit engineering boundaries. It documents the guardrails that kept the project local-first, scoped, layered, and disciplined.

Why This Matters

Many AI data demos upload files to hosted tools. This project shows a different path: keep data local, keep runs reproducible, separate deterministic data work from model narration, and preserve the evidence behind each answer.
