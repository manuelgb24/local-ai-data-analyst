Local AI Data Analysis Experiment

Can free local AI take over junior data analysis work?

A reproducible local-first experiment that tests how far DuckDB, Ollama, a single data_analyst agent, and traceable artifacts can go before human judgment becomes essential again.

The Experiment

This project asks a practical question: how much of the early junior analyst loop can a free local AI cover? It loads local CSV, XLSX, and Parquet files, profiles them with DuckDB, answers focused questions with a local model, and keeps the evidence behind each answer.

Where The Human Still Matters

The product does not claim that AI fully replaces analysts. It is strongest at first-pass exploration, simple questions, summaries, and traceable reporting. A human still owns the business question, data quality judgment, domain context, and final conclusion.

How It Works

  • Local dataset → CSV, XLSX, or Parquet file
  • DuckDB → local profiling and analytical queries
  • data_analyst agent → analytical reasoning layer
  • Ollama → free local LLM narration with deepseek-r1:8b
  • Artifacts → traceable results, tables, charts, and run outputs

Architecture Overview

Interfaces

Web UI, local API, and CLI share the same application core.

Runtime

Runs are coordinated through a clear runtime layer with local state and artifacts.

Adapters

DuckDB and Ollama are isolated behind adapters to keep the core clean.

Engineering Highlights

Local-first Free local model workflow No hosted backend DuckDB Ollama Reproducible demo dataset Traceable artifacts Tests

Why This Matters

Many AI data demos upload files to hosted tools or hide the evidence behind an answer. This project shows a different path: keep data local, use a free local model workflow, separate deterministic data work from model narration, and preserve the trail behind every conclusion.
